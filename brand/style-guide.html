Vantx Brand Style Guide

Reference for documents, presentations, proposals, and any branded material outside the website. All values match the web design system.

v1.0 — March 2026

Logo

The Vantx logo consists of a geometric hexagonal mark and the wordmark "VANTX" in DM Sans Bold. Always use the provided assets — never recreate or modify the logo.

Primary usage

White version — use on dark backgrounds (#1A1917 or darker).
Dark version — use on light backgrounds (#F8F7F4 or lighter). Currently rendered via CSS filter; a native dark asset (logomark in #1B6B4A, wordmark in #1A1A17) is pending.

On brand green

Clear space & minimum size

Maintain clear space equal to the height of the "V" letterform on all sides. Minimum width: 100px for digital, 25mm for print.

Logo don'ts

Don't
Rotate, skew, or distort the logo
Don't
Add drop shadows, gradients, or effects
Don't
Place on busy backgrounds or photos without overlay
Don't
Change the logo colors (only white or dark)

Pending: dark logo asset

The site currently only has a white logo. For light backgrounds it uses a CSS filter: brightness(0) hack that turns it solid black. You need a proper dark version: logomark in forest green (#1B6B4A) and wordmark in near-black (#1A1A17). Export as SVG + PNG at 1x, 2x, and 3x.

Color usage rules

Do
Use forest green (#1B6B4A) as primary accent for CTAs, links, highlights
Do
Use parchment (#F8F7F4) as the default background, not pure white
Don't
Use service colors (blue, teal, amber, purple) for general brand materials — they're for service-specific contexts only
Don't
Use cool grays, blue-grays, or pure black (#000). The Vantx palette is warm throughout

Typography rules

Do
Use negative letter-spacing on headings (-0.03 to -0.04em). Tight tracking signals premium.
Do
Use JetBrains Mono for any data: prices, metrics, percentages, code snippets
Don't
Use more than 2 font weights per slide/page. Stick to Regular (400) + Bold (700)
Don't
Use all-caps body text. Uppercase is reserved for labels and overlines only

Voice & Tone

Brand personality

Vantx sounds like a senior engineer who happens to be good at explaining things. Technical, direct, specific — but never condescending.

Do
Be specific. "CREATE INDEX idx_x ON settlements(merchant_id)" not "consider optimizing the database"
Do
Use data. "p95 latency dropped from 1.2s to 340ms" not "we improved performance significantly"
Do
Be direct. "Flat monthly rate. Cancel anytime." not "We offer flexible subscription options"
Do
Name the pain. "Alerts firing at 3 AM" not "Operational challenges"
Don't
Use hourly language. Never "per hour", "hourly rate", "X hours/month". The model is flat monthly subscription.
Don't
Use corporate filler: "leverage", "synergy", "best-in-class", "cutting-edge", "innovative solution"
Don't
Be vague. Every claim needs a number, an example, or a deliverable behind it.
Don't
Oversell. "We help your system handle more traffic" not "We 10x your revenue"
