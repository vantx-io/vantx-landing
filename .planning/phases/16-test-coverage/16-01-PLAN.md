2. Create `platform/__tests__/api/checkout.test.ts` with the following structure:

   **vi.mock declarations (BEFORE any route imports):**
   - `vi.mock('@/lib/supabase/server')` returning `{ createServiceClient: vi.fn(), createServerSupabase: vi.fn() }`
   - `vi.mock('@/lib/rate-limit')` returning `{ rateLimit: vi.fn().mockResolvedValue({ success: true, limit: 5, remaining: 4, reset: Date.now() + 60000 }), getRateLimitIdentifier: vi.fn().mockReturnValue('ip:test'), rateLimitResponse: vi.fn(), rateLimitHeaders: vi.fn().mockReturnValue({}) }`
   - `vi.mock('@/lib/stripe')` returning `{ getStripe: vi.fn().mockReturnValue({ customers: { create: vi.fn().mockResolvedValue({ id: 'cus_test_123' }) }, checkout: { sessions: { create: vi.fn().mockResolvedValue({ url: 'https://checkout.stripe.com/test-session' }) } } }) }`

   **Imports (AFTER mocks):**
   - `import { testApiHandler } from 'next-test-api-route-handler'`
   - `import * as appHandler from '@/app/api/checkout/route'`
   - `import { createServiceClient, createServerSupabase } from '@/lib/supabase/server'`
   - `import { getStripe } from '@/lib/stripe'`

   **beforeEach:** `vi.clearAllMocks()` + set up `createServerSupabase` mock to return `{ auth: { getUser: vi.fn().mockResolvedValue({ data: { user: { id: 'user-001' } }, error: null }) } }`

   **Test cases (per D-06):**

   a) `'returns 200 with checkout URL when customer exists'`:
      - Mock `createServiceClient` to return chainable `from('subscriptions').select().eq().single()` resolving `{ data: { stripe_customer_id: 'cus_existing' }, error: null }`
      - Call `testApiHandler({ appHandler, test: async ({ fetch }) => { ... } })` with POST body `{ clientId: 'client-001', priceId: 'price_test' }`
      - Assert: `res.status` is `200`, `json.url` is `'https://checkout.stripe.com/test-session'`

   b) `'creates Stripe customer when none exists and returns 200'`:
      - Mock `createServiceClient` with `.mockReturnValue` returning a `from` function that uses `vi.fn()`:
        - First call to `from('subscriptions').select().eq().single()` returns `{ data: null, error: null }` (no existing customer)
        - Second call to `from('clients').select().eq().single()` returns `{ data: { name: 'Acme', email: 'acme@test.com' }, error: null }`
        - Third call to `from('subscriptions').update().eq()` returns `{ error: null }`
      - Assert: `res.status` is `200`, `getStripe().customers.create` was called with `expect.objectContaining({ name: 'Acme', email: 'acme@test.com', metadata: { client_id: 'client-001' } })`

   c) `'returns 400 when req.json() throws'`:
      - Call with invalid JSON body or missing content-type
      - Assert: `res.status` is `400`, response body contains `error` field

   d) `'returns 429 when rate limited'`:
      - Override `rateLimit` mock: `vi.mocked(rateLimit).mockResolvedValueOnce({ success: false, limit: 5, remaining: 0, reset: Date.now() + 60000 })`
      - Override `rateLimitResponse` mock to return `new Response(JSON.stringify({ error: 'Rate limit exceeded' }), { status: 429 })`
      - Assert: `res.status` is `429`

   **Do NOT re-mock `next/headers`** -- NTARH handles this internally and `createServerSupabase` is already mocked at module level so `cookies()` is never called (Pitfall 1).
