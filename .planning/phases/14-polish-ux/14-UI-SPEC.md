## Spacing Scale

Standard 8-point scale. All new components must use these values only.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Skeleton inner gap, badge padding |
| sm | 8px | Skeleton row gap, inline spacing |
| md | 16px | Default element spacing, card padding inner sections |
| lg | 24px | Card padding (matches existing `p-6` = 24px) |
| xl | 32px | Section break between onboarding card and page content |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- `p-5` (20px) is used in existing MetricCard / StatCard — skeleton cards that mirror these must use 20px padding to match shape exactly
- Touch target minimum: 44px height for the onboarding dismiss button

## Typography

Four sizes only. New components introduced in this phase must not add a fifth size.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 24px (`text-2xl`) | 600 (`font-semibold`) | 1.2 | Page headings (`h1`) — new components only |
| Heading | 16px (`text-base`) | 600 (`font-semibold`) | 1.3 | Section headings (`h2`), card titles, onboarding card heading |
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | Body text, descriptions, input text, link items |
| Label | 12px (`text-xs`) | 600 (`font-semibold`) | 1.4 | Stat labels, table headers, badge text, timestamps, secondary badge text, inline messages |

Weight constraint: this phase introduces only 2 weights — 400 (regular) and 600 (semibold). Weight 700 (`font-bold`) exists in pre-existing `h1` and stat card markup; this phase does not modify those elements and does not introduce `font-bold` in any new component.

Size constraint: `text-lg` (18px) is not used in new components. Existing elements that rendered at 18px — such as section headings or card titles — are governed by Heading at 16px (`text-base`) in new components introduced by this phase.

## Component Specifications

### SkeletonCard

Mirrors the shape of existing `MetricCard` / `StatCard` components.

```
bg-white rounded-xl p-5 border border-gray-100 flex-1 min-w-[160px] animate-pulse
  ├── Label line: h-3 w-20 bg-gray-200 rounded mb-3
  ├── Value line: h-7 w-16 bg-gray-200 rounded mb-2
  └── Delta line: h-2.5 w-12 bg-gray-100 rounded
```

### SkeletonTable

Mirrors the shape of existing table rows (Tests, Admin tables).

```
bg-white rounded-xl border border-gray-100 animate-pulse
  ├── Header row: h-9 w-full bg-gray-200 rounded-t-xl mb-0
  └── 5x data rows:
        ├── Cell 1: h-4 w-32 bg-gray-200 rounded
        ├── Cell 2: h-4 w-16 bg-gray-100 rounded
        └── Cell 3: h-4 w-16 bg-gray-100 rounded
      Row separator: border-b border-gray-50, py-3 px-5
```

### SkeletonText

Mirrors a paragraph or list of text lines (Reports, Tasks list items).

```
animate-pulse space-y-2
  ├── Line 1: h-4 w-3/4 bg-gray-200 rounded
  ├── Line 2: h-3 w-1/2 bg-gray-100 rounded
  └── Line 3 (optional): h-3 w-2/3 bg-gray-100 rounded
```

### OnboardingCard

Dismissed permanently by setting `has_onboarded = true`.

```
bg-brand-accent/8 border border-brand-accent/20 rounded-xl p-6 mb-6
  ├── Heading: text-base font-semibold text-brand-dark — "Welcome to Vantx!" / "Bienvenido a Vantx!"
  ├── Subtext: text-sm text-brand-muted mb-4 — "Here are a few things to get you started." / "Aquí hay algunas cosas para comenzar."
  ├── Action list: flex flex-col gap-2
  │     ├── Link item: text-sm text-brand-accent font-medium hover:underline
  │     │     • "View your tasks →" / "Ver tus tareas →"
  │     │     • "Check test results →" / "Ver resultados de pruebas →"
  │     │     • "Explore billing →" / "Explorar facturación →"
  │     │     └── "Update notification settings →" / "Actualizar preferencias de notificación →"
  └── Dismiss button: px-4 py-2 mt-4 text-sm font-medium bg-brand-accent text-white rounded-lg
        hover:bg-brand-accent-hover transition — "Got it" / "Entendido"
        min-height: 44px (touch target)
```

Dismiss interaction:
1. User clicks "Got it" / "Entendido"
2. Button shows spinner (disabled state, opacity-70)
3. PATCH `/api/profile` with `{ has_onboarded: true }`
4. On success: card fades out with `transition-opacity duration-300 opacity-0`, then unmounts
5. On API failure: button re-enables, no error shown (silent fail — card stays visible, user can retry)

### SectionErrorBoundary

Class component wrapping individual data sections. Does NOT wrap an entire page.

```
Fallback UI:
bg-gray-50 border border-gray-200 rounded-xl p-6 text-center
  ├── Icon: text-2xl mb-2 — "⚠" (unicode, not lucide, to avoid import in class component)
  ├── Heading: text-sm font-semibold text-brand-dark mb-1
  │     EN: "Something went wrong in this section"
  │     ES: "Algo salió mal en esta sección"
  ├── Body: text-xs text-brand-muted mb-4
  │     EN: "This section couldn't load. The rest of the page is still available."
  │     ES: "Esta sección no pudo cargar. El resto de la página sigue disponible."
  └── Retry button: px-4 py-2 text-sm font-medium border border-gray-200 rounded-lg
        text-brand-muted hover:bg-white hover:text-brand-dark transition
        EN: "Try again"  ES: "Intentar de nuevo"
        onClick: this.setState({ hasError: false })
```

Retry interaction: calls `setState({ hasError: false })` to re-render children. No network call from the boundary itself — children re-run their own data fetch on mount.

## Interaction States

### Loading (skeleton phase)
- Skeleton renders immediately on mount (no delay)
- `animate-pulse` class drives the shimmer — CSS only, no JS timer
- `aria-busy="true"` on the section container while skeleton is shown
- `aria-label="Loading content"` on the skeleton wrapper div

### Onboarding card dismiss
- Disabled state: button `opacity-70 cursor-not-allowed`, shows inline spinner (border-2 border-white border-t-transparent rounded-full w-4 h-4 animate-spin inline-block mr-2)
- Success: `transition-opacity duration-300` fade to opacity-0, then `display: none` via state
- Failure: silent — button re-enables at full opacity, card stays visible

### Error boundary retry
- Button click calls `this.setState({ hasError: false })`
- Children re-mount and re-run their own useEffect data fetch
- No loading indicator on the boundary itself — child's own skeleton handles loading state

|--------|-------|-----------|
| Skeleton pulse | `animate-pulse` (Tailwind built-in, 2s ease-in-out infinite) | All skeleton components |

| Spinner | `animate-spin` | Dismiss button loading state |

No page transitions. No entry animations on skeleton-to-content swap. Content replaces skeleton instantly on state update.
