   async function handleForgotPassword(e: React.FormEvent) {
     e.preventDefault();
     setForgotLoading(true);

     await supabase.auth.resetPasswordForEmail(forgotEmail, {

**Reset-password page** (per D-04, D-05, D-06):

Create `platform/src/app/[locale]/reset-password/page.tsx` as a `"use client"` component:

1. Imports: `useState`, `useEffect` from react; `createClient` from `@/lib/supabase/client`; `useRouter` from `next/navigation`; `useLocale`, `useTranslations` from `next-intl`.

2. State: `ready` (boolean, false), `error` (string), `newPassword` (string), `confirmPassword` (string), `loading` (boolean), `success` (boolean).

3. Use `onAuthStateChange` (NOT `getSession`) to detect the recovery session (Pitfall 1 from research). On mount:
   ```typescript
   useEffect(() => {
     const supabase = createClient();
     const { data: { subscription } } = supabase.auth.onAuthStateChange((event) => {
       if (event === "PASSWORD_RECOVERY") {
         setReady(true);
       }
     });
     // Fallback: if no event after 3s, redirect to login
     const timeout = setTimeout(() => {
       if (!ready) {
         router.replace(`/${locale}/login`);
       }
     }, 3000);
     return () => {
       subscription.unsubscribe();
       clearTimeout(timeout);
     };
   }, []);
   ```
   NOTE: The `ready` ref check inside timeout needs a ref or the `ready` state check should be done via a ref to avoid stale closure. Use a `readyRef` with `useRef(false)` and set both `setReady(true)` and `readyRef.current = true` when PASSWORD_RECOVERY fires.

4. `handleReset` function:
   - Validate: `newPassword.length < 8` -> `setError(t("error_min_length"))` and return
   - Validate: `newPassword !== confirmPassword` -> `setError(t("error_mismatch"))` and return
   - Call `createClient().auth.updateUser({ password: newPassword })`
   - On error: `setError(error.message)`
   - On success: `setSuccess(true)`, then `setTimeout(() => router.push(\`/\${locale}/login?reset=success\`), 1500)`

5. Render: Same visual style as login page (centered card, brand colors). Use `t = useTranslations("reset_password")`.
   - While `!ready`: show a loading spinner/text `t("loading")` -- "Verifying reset link..."
   - While `ready && !success`: show form with heading `t("title")` ("Set new password"), two password inputs (new + confirm), error display, submit button
   - While `success`: show success message `t("success")` ("Password updated! Redirecting to login...")
   - Card styling: same as login -- `bg-brand-surface border border-gray-200/60 rounded-2xl p-8 shadow-sm` with `max-w-[420px]`
   - Input styling: same as login -- `w-full px-3.5 py-2.5 rounded-lg border border-gray-200 bg-white text-brand-dark text-sm`
   - Button styling: same as login -- `w-full h-11 bg-brand-accent hover:bg-brand-accent-hover text-white font-medium rounded-lg transition disabled:opacity-50`
