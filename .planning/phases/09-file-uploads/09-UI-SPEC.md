## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (px-1, gap-1) |
| sm | 8px | Compact element spacing (px-2, py-2, gap-2) |
| md | 16px | Default element spacing (px-4, gap-4) |
| lg | 24px | Section padding (px-6, py-6) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch target minimum for paperclip button: 44px (use `min-w-[44px] min-h-[44px]` with centered icon)
- Image thumbnail size: 120×120px (locked from D-05, not a spacing token — explicit `w-[120px] h-[120px]`)
- Progress bar height: 4px (`h-1`)
- Notification badge height: 18px (existing pattern from NotificationBell — not a standard token)

Source: Existing tasks page and NotificationBell patterns; D-05 in 09-CONTEXT.md

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label / Badge | 10px (text-[10px]) | 600 (semibold) | 1.2 |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 |
| Meta / Timestamp | 12px (text-xs / text-[12px]) | 400 (regular) | 1.4 |

Notes:
- Task titles use 14px semibold (600) — same size as body but different weight. This is an established pattern in tasks/page.tsx and must be maintained for the comment/attachment display area.
- File size text (e.g., "2.3 MB") uses 12px regular — same as meta/timestamp role.
- File type badge text (e.g., "PDF", "HAR") uses 10px semibold — same as existing Badge component in tasks/page.tsx.
- No new font sizes introduced by this phase. Work within the 4 sizes above.

Source: `platform/src/app/[locale]/portal/tasks/page.tsx` — existing className patterns

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FDFCFA (brand-surface) | Page background, comment thread background |
| Secondary (30%) | #F8F7F4 (brand-bg) + white (#FFFFFF) | Comment section bg (`bg-gray-50`), individual comment cards (`bg-white`), file preview strip |
| Accent (10%) | #1B6B4A (brand-accent) | Paperclip button active state, progress bar fill, focus ring on comment input, primary "Send" button |
| Destructive | #DC2626 (brand-red) | Inline file rejection error text only (e.g., "file.exe — executables not allowed") |

Accent reserved for:
1. Progress bar fill color during upload
2. Focus border on comment textarea (`focus:border-brand-accent`)
3. "Send" button background (`bg-brand-accent text-white`)
4. Drag-over border highlight on comment area (`border-brand-accent`)
5. Paperclip icon color when files are staged (active state indicator)

NOT accent: drag-over overlay background (use `bg-brand-accent/5` — very low opacity), file type badges (use per-type semantic colors matching existing Badge pattern), lightbox overlay (`bg-black/70`).

Additional semantic colors (matching existing tasks page pattern — not new introductions):
- Upload progress track: `bg-gray-200` (inactive track)
- File card border: `border-gray-100` (matches existing comment card border)
- Muted text on file metadata: `text-gray-400` / `text-brand-muted` (#5C5C56)
- File type badge colors (reuse existing badge color map pattern from tasks page):
  - Images: `#27AE60` (brand-green)
  - Documents (PDF, DOCX): `#2E75B6`
  - Archives (ZIP, GZ): `#8E44AD`
  - Data/config (JSON, YAML, CSV): `#E67E22`
  - HAR/log files: `#5C5C56` (brand-muted)
  - Other: `#999`

Source: `platform/tailwind.config.ts`, `platform/src/app/[locale]/portal/tasks/page.tsx` Badge component

## Component Inventory

This phase introduces new UI elements. Each must be implemented as a self-contained unit within `tasks/page.tsx` or extracted as a component file.

### 1. Comment Form with File Input

**Location:** Replaces the existing `<div className="flex gap-2 mt-3">` comment input area in tasks/page.tsx

**Layout:**
```
[ Textarea (multiline, flex-1)          ]
[ Preview Strip (conditional, if files) ]
[ Paperclip btn ]         [ Send btn    ]
```

- Textarea: `w-full px-3 py-2 rounded-lg border border-gray-200 text-sm focus:border-brand-accent focus:outline-none min-h-[80px]` — replaces single-line input
- Drag-over state: add `border-brand-accent bg-brand-accent/5` classes to the textarea wrapper div
- Paperclip button: `p-2 rounded-lg hover:bg-gray-100 transition min-w-[44px] min-h-[44px] flex items-center justify-center` — use `Paperclip` icon from lucide-react (w-4 h-4 text-brand-muted); active state (files staged): `text-brand-accent`
- Hidden file input: `<input type="file" multiple accept="..." className="hidden" />`
- Send button: existing pattern `px-4 py-2 rounded-lg bg-brand-accent text-white text-sm font-semibold`

### 2. File Preview Strip

**Trigger:** Appears between textarea and the button row when `pendingFiles.length > 0`

**Container:** `flex flex-wrap gap-2 py-2`

**Image preview item** (for image/* MIME types):
- Outer: `relative w-[80px] h-[80px] rounded-lg overflow-hidden border border-gray-200 flex-shrink-0`
- `<img>` at `object-cover w-full h-full`
- Remove button: `absolute top-0.5 right-0.5 w-5 h-5 rounded-full bg-black/50 flex items-center justify-center hover:bg-black/70 transition` with `X` icon (w-3 h-3 text-white from lucide-react)
- Progress bar (during upload): `absolute bottom-0 left-0 right-0 h-1 bg-gray-200`; fill div `h-1 bg-brand-accent transition-all`
- Error overlay: `absolute inset-0 bg-red-500/20 flex items-center justify-center`; error icon `AlertCircle w-5 h-5 text-red-500`

Note: Preview strip uses 80×80px thumbnails (compact for the form). The 120×120px size (D-05) applies to the **comment thread display**, not the pre-upload preview strip.

**Non-image file item:**
- Container: `flex items-center gap-2 px-2 py-1.5 rounded-lg border border-gray-100 bg-white max-w-[200px]`
- File icon: `FileText` / `File` from lucide-react (w-4 h-4 text-brand-muted)
- Filename: `text-xs text-brand-dark truncate flex-1`
- Remove `x` button: `w-4 h-4 text-gray-400 hover:text-gray-600 flex-shrink-0` using `X` icon
- Progress bar below the entire item: `h-1 rounded-full bg-gray-200 mt-1`; fill div `h-1 rounded-full bg-brand-accent`

**Inline rejection error** (replaces item or appears below it):
- `text-xs text-brand-red` — e.g., "script.exe — executables not allowed"
- `text-xs text-brand-red` — e.g., "video.mp4 — exceeds 50 MB limit"

### 3. Attachment Display in Comments (Thread View)

**Location:** Inside each comment card, below `<div className="text-sm text-gray-700">{c.content}</div>`

**Image grid** (when comment has image attachments):
- Container: `flex flex-wrap gap-2 mt-2`
- Each thumbnail: `w-[120px] h-[120px] rounded-lg overflow-hidden border border-gray-100 cursor-pointer hover:opacity-90 transition flex-shrink-0`
- `<img>` at `object-cover w-full h-full` — uses signed URL
- On click: opens lightbox (see component 4)

**Non-image file list** (when comment has non-image attachments):
- Container: `flex flex-col gap-1.5 mt-2`
- Each file row: `flex items-center gap-2 px-3 py-2 rounded-lg border border-gray-100 bg-brand-bg hover:bg-gray-100 transition cursor-pointer`
- File icon: `FileText` / `File` / `Archive` from lucide-react (w-4 h-4 text-brand-muted)
- Filename: `text-sm text-brand-dark flex-1 truncate`
- File size: `text-xs text-gray-400 flex-shrink-0`
- Type badge: reuse existing `Badge` component from tasks/page.tsx with per-type colors listed in Color section above
- Download icon: `Download` from lucide-react (w-4 h-4 text-brand-muted ml-1 flex-shrink-0) — appears on hover

**Mixed layout order:** Image grid always renders first, file list below it.

### 4. Lightbox / Image Modal

**Trigger:** Click on any image thumbnail in the comment thread

**Implementation:** Simple overlay — no library. Pure React state + portal or absolute positioning.

**Overlay:** `fixed inset-0 bg-black/70 z-50 flex items-center justify-center` — click outside closes
**Image:** `max-w-[90vw] max-h-[90vh] object-contain rounded-lg shadow-2xl`
**Close button:** `absolute top-4 right-4 p-2 rounded-full bg-black/40 hover:bg-black/60 transition`; `X` icon (w-5 h-5 text-white)
**Keyboard:** Escape key closes (follow NotificationBell pattern for keydown listener + cleanup)
**Body scroll lock:** Add `overflow-hidden` to `document.body` on open, remove on close

All copy must exist in both `en.json` and `es.json` under the `tasks` namespace. New keys listed below.

| Element | English Copy | Key |
|---------|-------------|-----|
| Primary CTA (file attach) | "Attach file" | tasks.attach_file |

| Send button | "Send" | tasks.send (existing — no change) |
| Drag-over prompt | "Drop files here" | tasks.drop_files_here |
| Empty attachment state (no files yet staged) | (no copy — strip hidden) | — |
| File rejection: executable blocked | "{filename} — executables not allowed" | tasks.error_executable |

- Comment submit fails: "Failed to post comment. Please try again." (`tasks.error_comment_submit`)
- Upload fails mid-progress: Show `AlertCircle` icon on the affected file thumbnail in the strip. Do not block submission of other files. Copy: "Upload failed — try again" (`tasks.error_upload_failed`)

## Accessibility Requirements

- Paperclip button: `aria-label={t('tasks.attach_file')}`, keyboard-accessible (`type="button"`)
- Hidden file input: associated to paperclip button via `onClick` trigger from button
- Comment textarea: existing `placeholder` is sufficient; no additional aria needed
- Drag-over: `aria-live="polite"` region for screen reader announcement of "Drop files here"
- Progress bars: `role="progressbar" aria-valuenow={percent} aria-valuemin={0} aria-valuemax={100}` on the fill div; `aria-label="Uploading {filename}"`
- Image thumbnails (thread): `alt={t('tasks.attachment_alt', { filename })}` — not empty alt
- Lightbox: `role="dialog" aria-modal="true" aria-label={t('tasks.close_preview')}`; focus trap on open; return focus to triggering thumbnail on close; Escape key closes (follow NotificationBell keydown pattern)
- File rejection errors: `role="alert"` on the error text so screen readers announce immediately
- Download buttons: `aria-label={t('tasks.download_file', { filename })}`
