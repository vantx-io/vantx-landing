## Typography

All type uses `--font-sans` (DM Sans). Monospace (`--font-mono` JetBrains Mono) is reserved strictly for pricing figures, data stats, and code abbreviations — never for prose labels.

| Role | CSS Variable | Size | Weight | Line Height | Letter Spacing |
|------|-------------|------|--------|-------------|----------------|
| Body | `--text-base` | 16px | `--font-normal` (400) | `--leading-normal` (1.6) | `--tracking-normal` (0) |
| Label / eyebrow | `--text-sm` | 14px | `--font-semibold` (600) | 1 | `--tracking-wider` (0.08em) |
| Heading (section) | `--text-3xl` / clamp(1.75rem, 4vw, 3rem) | 30px–48px fluid | `--font-bold` (700) | `--leading-tight` (1.15) | `--tracking-snug` (-0.02em) |
| Display (hero h1) | `--text-5xl` / `--text-6xl` | 48px–60px | `--font-bold` (700) | `--leading-display` (1.05) | `--tracking-tight` (-0.03em) |

Active weights in use: regular 400 + bold 700. Semibold 600 reserved for labels/eyebrows only.

Sub-body / meta text: `--text-xs` (12px) at `--font-semibold` (600) for badges, tags, overlines.

**Source:** `landing/css/base.css` lines 27–29, 94–95, 165–166, 173–176, `.impeccable.md` typography rules

## Color

Light mode (`:root`) and dark mode (`@media (prefers-color-scheme: dark)`) are both in scope for Phase 4 — WCAG 2.1 AA compliance must pass in both modes.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) — bg | `--color-bg` #F8F7F4 warm parchment | `--color-bg` #1A1917 warm dark | Page background on all pages |
| Secondary (30%) — surfaces | `--color-surface` #FDFCFA | `--color-surface` #242320 | Cards, panels, nav, detail page sections |
| Secondary — elevated | `--color-surface-elevated` #F0EFE9 | `--color-surface-elevated` #2E2D28 | Hover states, dropdowns |
| Accent (10%) — forest green | `--color-accent` #1B6B4A | `--color-accent` #259B6E | CTAs, key data, accent headline — ONLY |
| Destructive | `--color-danger` #DC2626 | `--color-danger` #EF4444 | No destructive actions in Phase 4 |

Accent reserved for:
- Primary CTA buttons ("Book a Free Call", "Schedule a Call")
- Section accent headlines (e.g. highlighted word in hero)
- Focus ring border (`--color-border-focus`)
- Active/current nav indicator

Accent is NOT used for: body text color, secondary buttons, decorative icons, section backgrounds, or border separators.

Service accent colors (not the primary brand accent):
- Observability: `--color-obs` teal-600
- Performance: `--color-perf` blue-600
- QA Tech Lead: `--color-qa` purple-600
These are reserved for service-specific badges, borders, and subtle backgrounds — never for CTA buttons.

WCAG constraint: All text tokens must pass 4.5:1 contrast against their background token. Fix path is `tokens.css` first (global), per-element overrides only if global change breaks something else (D-08).

**Source:** `landing/css/tokens.css` full file, CONTEXT.md D-06/D-07/D-08, `.impeccable.md` design principles

## Interaction Contracts

### Credibility Bar Entrance (D-04 fix)

Current: `animation: hero-enter 600ms var(--ease-out) 700ms both` — 700ms delay causes pop-in.

**Contract:** Replace with fade-only, no delay:
```css
.credibility {
  animation: fade-in 400ms var(--ease-out) both;
}
@keyframes fade-in {
  from { opacity: 0; }
}
```
Removes the translateY that causes the pop-in while maintaining a gentle entrance. The `hero-enter` keyframe is kept intact for other uses.

### Hero Height (D-05 fix)

Current: `min-height: calc(100dvh - var(--nav-height))` creates empty whitespace on desktop.

**Contract:** Desktop override at `min-width: 1024px`:
```css
@media (min-width: 1024px) {
  .hero {
    min-height: 0;
    padding-top: var(--space-24);
    padding-bottom: var(--space-24);
  }
}
```
Mobile rule (`min-height: 100dvh`) is preserved — it is correct for mobile browser chrome.

### Dashed Border Fix

**Contract:** Change `.svc-card__onetime` border from `dashed` to `solid`:
```css
.svc-card__onetime {
  border-top: 1px solid var(--color-border);
}
```
This is the only dashed border on the site — it is an outlier against all other borders.

### CLS Fix — content-visibility Sections (D-15 fix)

**Contract:** Measure actual rendered heights in Chrome DevTools and update these `contain-intrinsic-size` values:
- `.compare` (landing.css ~line 192): `contain-intrinsic-size: auto Xpx` — measure `.compare` computed height
- `.pain` (landing.css ~line 344): `contain-intrinsic-size: auto Xpx` — measure `.pain` computed height
- `.faq` (base.css ~line 699): `contain-intrinsic-size: auto Xpx` — measure `.faq` computed height
- Third `content-visibility: auto` section (landing.css ~line 829): measure and update

Do NOT remove `content-visibility: auto` — it is a performance optimization. Only calibrate the intrinsic-size values.

### "Also Available" CTA Row (D-16 fix)

**Contract:** Remove the per-row "Book a Call" buttons entirely. Replace the entire `services__also-row` CTA pattern with a single anchor at the bottom of the "also available" group:

```
"See all services →" (text link, not a button — low visual weight)
```

Per-row CTAs read as template artifact. A single text link at the group end is cleaner and less conversion-competing with the primary CTA.

### Calendly Popup Focus (D-10 verification)

**Contract:** Verify (do not implement — Calendly controls this):
- Tab key does not reach page elements behind the open Calendly overlay
- Escape key closes the popup
If Calendly's own focus trap is broken, document as a known third-party limitation. Do NOT add a wrapper focus trap around `Calendly.initPopupWidget()` — it will cause a double-trap conflict.

### Mobile Nav Drawer Focus (D-11 verification)

**Contract:** Verify that the existing focus trap in `partials.js` lines 127–142:
- Traps Tab key within the open drawer (no focus escapes to page)
- Escape key closes the drawer and returns focus to the hamburger button
- `aria-expanded` toggles correctly on the hamburger button

### LATAM Banner Alignment (from todo.md)

**Contract:** Change `text-align: center` on `.latam-banner` to `text-align: left` to match the left-aligned page rhythm. The banner currently stands out as the only centered prose block on the page.

## Accessibility Contract (Phase 4 requirements)

These are verification requirements, not implementation requirements. The code already has the right patterns — this phase confirms they work correctly.

| Check | Standard | Method | Scope |
|-------|----------|--------|-------|
| Contrast — light mode | WCAG 2.1 AA (4.5:1 for normal text, 3:1 for large text) | axe DevTools browser extension | All 8 pages |
| Contrast — dark mode | WCAG 2.1 AA (same thresholds) | axe DevTools + Chrome Rendering tab: force `prefers-color-scheme: dark` | All 8 pages |
| Keyboard navigation | Tab order follows visual reading order; no keyboard traps outside intentional dialogs | Manual tab-through | All 8 pages |
| Screen reader labels | All interactive elements have accessible names; landmark regions present | axe DevTools + VoiceOver spot check | All 8 pages |
| Focus visible | 2px solid focus ring using `--color-border-focus` on all focusable elements | Visual inspection during tab-through | All 8 pages |
| Calendly popup | Focus stays within overlay when open; Escape closes it | Manual keyboard test | index.html + detail pages |
| Mobile nav drawer | Tab cycles within open drawer; Escape closes drawer; hamburger aria-expanded toggles | Manual keyboard test | All pages with nav |

**Contrast fix protocol (D-08):** When axe flags a contrast failure, identify the `--color-*` token used by the element, fix the token value in `tokens.css` in both `:root` (light) and `@media (prefers-color-scheme: dark) :root` (dark) blocks simultaneously. All components using the token inherit the fix automatically.
